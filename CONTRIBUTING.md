# Contributing to InnoForge

Thanks for your interest in InnoForge! This project is being built in the open, and contributions of all sizes are welcome.

## Ways to contribute

- 💡 **Ideas & feedback** — open an issue and tell us what you'd want from a "build-anything management" platform.
- 🐛 **Bug reports** — found something broken? Open an issue with steps to reproduce.
- 🔧 **Code** — pick up an open issue, or open a PR with an improvement.
- 📖 **Docs & translations** — help make InnoForge approachable, especially in English and Arabic.

## Getting started

1. Fork the repository.
2. Create a branch: `git checkout -b feature/my-change`.
3. Make your change and commit it with a clear message.
4. Push and open a pull request describing what you changed and why.

## Ground rules

- Be kind and constructive.
- Keep pull requests focused — one logical change per PR is easiest to review.
- Explain the *why*, not just the *what*.

No contribution is too small. Welcome aboard. 🔨
